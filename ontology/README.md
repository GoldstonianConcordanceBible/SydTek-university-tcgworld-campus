# GCB Ontology Layer

The ontology layer defines the conceptual structure of the Goldstonian Concordance Bible (GCB) ecosystem.

This layer organizes:

• biblical texts  
• doctrines  
• theological entities  
• educational structures  
• interpretive frameworks  

into a structured system that both humans and machines can understand.

---

## Why This Exists

Most Bible study systems are:

• text collections
• commentaries
• video libraries

The GCB ecosystem aims to go further.

It aims to become a **structured theological knowledge graph**.

That means every concept can eventually be represented as:

• an entity
• a relationship
• a property
• a reference

---

## Core Ontology Domains

The ontology layer organizes knowledge into several domains.

### Canon Domain

Entities include:

• books of the Bible  
• canon collections  
• manuscript traditions  
• canonical categories  

Primary anchor:

**81-Book Ethiopian Canon**

---

### Doctrine Domain

Entities include:

• doctrines
• theological frameworks
• interpretive models

Primary anchor:

**Mirror → Water → Fire Doctrine**

---

### Entity Domain

Entities include:

• angels
• demons
• prophets
• apostles
• historical figures
• symbolic entities

Examples:

• seraphim  
• cherubim  
• watchers  
• archangels  

---

### Narrative Domain

Entities include:

• events
• locations
• covenants
• kingdoms
• prophetic sequences

Examples:

• Exodus
• Babylonian exile
• crucifixion
• resurrection

---

### Educational Domain

Entities include:

• programs
• pathways
• playlists
• NPC instructors
• academic structures

Primary anchor:

**SydTek University / SydTek Scholars**

---

## Ontology Structure

Each ontology entry should include:

• entity_id
• name
• type
• description
• relationships
• references

This allows the system to grow without breaking structure.
